The Semantic Way: 6 keys to success

1. Build main structures with semantic elements

Use semantic elements like:

  • <header> to identify the header of your page.
  • <nav> to create navigation links.
  • <main> to allocate the main content of your page.
  • <section> to create sections for your page.
  • <aside> to create sidebars.
  • <article> to create articles.
  • <footer> to identify the footer of your page.

to structure your HTML.

2. Use the right attributes

Use the right attributes like:

  • id to identify unique elements
  • class to group related elements
  • href to link to other pages

3. Use the right heading levels

Every page must only contain one <h1> element.
Use <h2> for subheadings, <h3> for sub-subheadings, and so on.

4. Dominate the difference between <article> , <section> and <div>

Use <article> for independent, self-contained content (like blog posts, news, etc.).
Use <section> to group related content (like every key listed here).
Use <div> to group generic content or to style elements.

<section> content must be divided into <div> .

5. Add semantics to your multimedia content

Use <audio> for audio content.
Use <video> for video content.
And use <figure> for figure content (like images, tables, code, etc.).

6. Froms also have some tags too

Always use <form> to create forms. Along with <fieldset> and <legend> to create a form fieldset (wich means a group of related form fields).
Also, make sure to always pair a <label> with every <input> .
